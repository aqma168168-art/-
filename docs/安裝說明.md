# 安裝說明

## 系統概覽

本系統分為兩個入口：

| 頁面 | 用途 | 安全機制 |
|------|------|----------|
| `login.html` | 統一登入頁，選擇身份 | — |
| `staff.html` | 員工作業入口（配發、回報） | 無需登入，只顯示作業資料 |
| `owner.html` | 老闆管理入口（損益、設定） | PIN 驗證，session 8 小時 |

---

## 第一步：建立 Google Sheets

1. 前往 [Google Sheets](https://sheets.google.com) 建立新試算表
2. 命名為「**中央廚房管理系統**」
3. **不需要手動建立分頁** — 執行 Apps Script 的 `initSheets()` 函式會自動建立全部工作表並填入預設資料

---

## 第二步：建立 Google Apps Script

1. 在 Google Sheets 中點選選單：**擴充功能 → Apps Script**
2. 刪除預設的空白程式碼
3. 將 `apps-script/Code.gs` 的內容貼上
4. 點選左側檔案選單，新增 `appsscript.json`（專案設定），貼上 `apps-script/appsscript.json` 的內容
5. 儲存（Ctrl+S）

### 初始化工作表

1. 在 Apps Script 編輯器頂端，函式下拉選單選擇 `initSheets`
2. 點擊「執行」
3. 第一次執行需授予權限，點擊「審查權限」→「允許」
4. 確認執行完成，回到 Google Sheets 檢查是否出現所有分頁

---

## 第三步：設定老闆 PIN

1. 在 Google Sheets → **Settings** 工作表
2. 新增一列：

   | key | value | description |
   |-----|-------|-------------|
   | `owner_pin` | `123456` | 老闆 PIN（建議改成 6 位數字） |

3. **請務必修改成自己的 PIN，不要使用預設值**

---

## 第四步：部署 Apps Script 為 Web App

1. 在 Apps Script 編輯器點選右上角「**部署**」→「**新增部署**」
2. 選擇類型：**網頁應用程式**
3. 設定：
   - **說明**：中央廚房管理系統 v1.0
   - **以以下身分執行**：我（你的 Google 帳戶）
   - **誰可以存取**：所有人（含匿名）
4. 點擊「部署」，複製產生的 **Web App URL**

URL 格式類似：
```
https://script.google.com/macros/s/AKfyc.../exec
```

---

## 第五步：設定前端 API URL

開啟 `js/config.js`，將 `APPS_SCRIPT_URL` 替換成你的 Web App URL：

```javascript
const CK_CONFIG = {
  APPS_SCRIPT_URL: 'https://script.google.com/macros/s/你的ID/exec',
  // ...
};
```

---

## 第六步：部署網站

### 方案 A：GitHub Pages（免費，推薦）

1. 建立 GitHub 帳號（若尚無）
2. 建立新的 repository（可設為 Private）
3. 上傳所有專案檔案（`index.html`、`staff.html`、`owner.html`、`login.html`、`css/`、`js/` 資料夾）
4. 前往 repository Settings → Pages
5. Source 選擇 **Deploy from a branch**，Branch 選 `main`，資料夾選 `/ (root)`
6. 儲存，等約 1 分鐘，系統網址格式為：`https://你的帳號.github.io/repository名稱/login.html`

### 方案 B：自己的網域

將所有檔案上傳至主機的 `public_html` 或 `www` 目錄即可，不需後端環境。

---

## 第七步：測試

1. 開啟 `login.html`
2. 確認底部顯示「系統連線正常」（綠點）
3. 選擇「員工作業」→ 確認可進入 `staff.html`
4. 回到 `login.html`，選擇「老闆管理」→ 輸入 PIN → 確認進入 `owner.html`
5. 在員工頁試填一筆配發記錄，確認寫入 Google Sheets

---

## 常見問題

**Q：登入頁顯示「無法連線」**
- 確認 `js/config.js` 的 `APPS_SCRIPT_URL` 已正確填入
- 確認 Apps Script 已部署（Deploy as Web App）且存取設定為「所有人」

**Q：Apps Script 執行失敗**
- 確認工作表名稱與 `Code.gs` 頂端的 `SHEETS` 常數完全相符（大小寫相同）
- 重新執行 `initSheets()` 確認全部工作表存在

**Q：如何修改 PIN？**
- 直接在 Google Sheets → Settings 工作表修改 `owner_pin` 的 value 欄位
- 不需要重新部署或修改程式碼

**Q：如何新增攤位？**
- 在 Google Sheets → StallConfig 工作表新增一列
- 填入 `stall_id`（如 S004）、`stall_name`、`active`（TRUE）、`rent`、`fixed_cost`
- 系統下次載入時自動讀取，不需修改任何程式碼

**Q：如何新增配料？**
- 在 Google Sheets → IngredientConfig 工作表新增一列
- 填入所有欄位，`active` 設為 TRUE

---

## 安全說明

- 老闆 PIN 僅透過 Apps Script 在後端驗證，前端不持有 PIN 明文
- `owner.html` 每個頁面頂端呼叫 `AUTH.requireOwner()`，未登入會立即導回 `login.html`
- Session 存在 `sessionStorage`（分頁關閉即自動登出），有效期 8 小時
- 員工頁面完全沒有任何成本、毛利、淨利的計算或顯示邏輯
